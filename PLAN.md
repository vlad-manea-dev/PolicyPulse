## Problem Statement

Build a hackathon project (4-hour timebox) that scores all current Irish TDs (Teachta Dála)
on political consistency — surfacing contradictions in their own statements over 20 years of
Dáil records AND, where possible, aligning stated positions against actual voting record.

Data sources:
- **Oireachtas API** — structured data: members, parties, votes, questions
- **KildareStreet API** — speech content: `getDebates`, `getHansard`, `getWrans`

## What Makes This Cool

20 years of every word spoken in the Dáil, plus voting records, plus an LLM that finds
the most embarrassing contradictions. The score isn't "a journalist said so" — it's "you
said the opposite thing in 2015." Objective, sourced, and deeply Irish.

## Constraints

- 4 hours total build time
- ~5 TDs per party (covering all major parties: Fianna Fáil, Fine Gael, Sinn Féin,
  Labour, Social Democrats, Green Party, plus key independents)
  - Selection criteria: pick the 5 longest-serving current members per party — more historical data,
    more likely to have contradictions. Use Oireachtas member data to sort by first_elected date.
- Next.js + React frontend
- Claude API for contradiction detection
- KildareStreet requires API key (free for non-commercial / hackathon use)
- Must be demo-able at end of session — pre-compute scores before demo

## Premises

1. Consistency scoring (TD vs. their own past statements) is more tractable and
   defensible than external fact-checking in a 4-hour window — agreed.
2. ~5 TDs per major party gives balanced party coverage without blowing the API
   budget or batch time — agreed.
3. Build something impressive; Claude is the tool, not the feature — agreed.

## Approaches Considered

### Approach A: Contradiction Finder
  Summary: Fetch each TD's debate history from KildareStreet, pass to Claude,
           surface top 3 contradiction pairs (quote A vs quote B, with dates).
  Effort:  M
  Risk:    Low — Claude output is the core value, minimal UI needed
  Pros:    Reliable, demo-able with one TD if pressed for time
  Cons:    Score is LLM-derived — subjective, occasionally wrong

### Approach B: Say vs. Vote Alignment
  Summary: Correlate speech content with Oireachtas /votes data to find cases
           where a TD's stated position contradicts how they voted.
  Effort:  L
  Risk:    High in isolation — aligning topics across two data sources is hard
  Pros:    Most defensible "truth" metric — binary vote record is objective
  Cons:    Topic-to-vote alignment needs careful prompt engineering

### Recommended: A+B Hybrid
  Core is Approach A (contradiction pairs). For each TD, also run a lightweight
  vote check on 2-3 pre-selected high-signal topics (housing, health, climate)
  where votes are easy to identify by bill name. If vote correlation takes too
  long, ship A clean — it stands on its own.

## Recommended Approach

**A+B Hybrid — contradiction pairs as the core, vote data as the upgrade.**

### Score Calculation

```
consistency_score = weighted average of:
  - contradiction_score (0–100): Claude rates severity of found contradictions
  - say_vs_vote_score (0–100): % of identifiable votes that match stated position
    (only included if vote data was successfully fetched; otherwise excluded)
```

### UI Flow

```
1. All Parties screen
   ├── Grid of party cards (logo, name, avg consistency score across their TDs)
   └── Click → Party screen

2. Party screen
   ├── Party name + header
   ├── Grid of TD cards (photo, name, constituency, consistency score badge)
   └── Click → Politician screen

3. Politician screen
   ├── Header: name, party, constituency, terms served
   ├── Consistency score (large, prominent — colour-coded: green/amber/red)
   ├── Top 3 Contradiction Pairs
   │   └── Each: Quote A (year) vs Quote B (year), topic label, severity badge
   └── Say vs Vote (if data available)
       └── 2-3 specific vote alignments / misalignments with bill name
```

### Build Order (4 hours)

**Hour 1 — Data pipeline (60 min, includes 15-min buffer)**
- Pre-flight checks (max 10 min)
- Hit Oireachtas `/members` + `/parties` to build the TD roster (5/party, ~35 TDs total)
- Hit KildareStreet `getDebates` for each TD — fetch 50 most recent + 50 from 5–10 years ago
  - Use **concurrent fetching with limit of 5** (use `p-limit` or manual queue): ~7x faster than sequential
  - Add 200ms delay between each request within each concurrent slot
  - On 429 or empty response: double the delay and retry once, then write empty file and move on
  - If rate-limited hard: reduce to 25 statements per TD — quality degrades gracefully
- Statement splitting: split raw debate text by paragraph (double newline). Filter out paragraphs < 30 words.
  These are the "statements" sent to Claude.
- Cache all raw data to `data/raw/{td-id}.json` — never re-fetch during demo
- 15-minute buffer for API errors, auth issues, or join failures

**Hour 2 — Claude processing (60 min)**
- Model: **claude-haiku-4-5** — ~$1 total, ~8-10 min for 35 TDs (well within $20 limit)
- Pre-filter: discard statements under 30 words before sending (removes procedural one-liners)
- ~35 TDs × 1 Claude call each ≈ 8–10 min at standard rate limits
- For each TD: run contradiction detection prompt, store to `data/scores/{td-id}.json`
  - Wrap each call in try/catch — on failure, write `{error: true, consistency_score: null}` and continue
  - Use `JSON.parse` on the Claude response; if it throws, log the raw response and skip that TD
  - Validate response shape: if `contradictions` array has <1 item or `consistency_score` is not a number,
    retry once with a simplified prompt ("Find at least 1 contradiction..."). If retry fails, write error placeholder.
  - After the batch, print a summary: "Scored X/35 TDs successfully"
- If 30+ min remain: for pre-selected bill vote IDs, hit Oireachtas `/votes` per TD and
  ask Claude to align statements — store in same score file
- If time is tight: skip vote alignment. Contradiction pairs alone are fully demo-able.

**Hour 2b — Merge step (5 min, end of Hour 2)**
- Run `scripts/build-index.ts` — merges `data/raw/` + `data/scores/` + roster into
  a single `data/all-scores.json` file keyed by party and TD
- Format: `{parties: [{name, slug, tds: [{id, name, photo, score, contradictions, say_vs_vote}]}]}`

**Hour 3 — Frontend (Next.js, 50 min)**
- Load `data/all-scores.json` via `fs.readFileSync` in `getStaticProps` — single read, zero runtime API calls
- All Parties screen: party grid with avg consistency scores
- Party screen: TD grid with score badges
- Politician screen: contradiction pairs + say/vote section (show section only if data exists)
  - Quotes: truncate display to 200 chars with "..." — store full quote in data, truncate in UI only
  - Failed TDs: filter out of party screen entirely; show count "Showing X/5 TDs for this party"

**Hour 4 — Polish + demo prep (60 min)**
- Fill vote data gaps if score file is missing say/vote section
- Responsive layout check
- Prepare demo script: pre-identify 2 TDs with the most dramatic contradictions (check
  `data/scores/` for highest severity contradiction pairs)

- KildareStreet `getDebates` params: `person=<id>`, `num=50`, `output=js`
- Oireachtas base URL: `https://api.oireachtas.ie/v1/`
- Oireachtas `/members` params: `chamber=dail`, `house_no=34` (34th Dáil — seated Jan 2025)

### Claude Prompt (contradiction detection)

```
You are analyzing speeches made by {td_name}, an Irish politician, in the Dáil.

Below are statements they made at different times. Find the top 3 most significant
contradictions — where they clearly stated opposite or incompatible positions on the
same topic at different points in time.

For each contradiction return:
{
  "topic": "one-word topic label",
  "quote_a": "exact quote",
  "quote_a_date": "YYYY-MM-DD",
  "quote_b": "exact quote",
  "quote_b_date": "YYYY-MM-DD",
  "severity": 1-3,  // 1=mild, 2=significant, 3=stark
  "explanation": "one sentence explaining the contradiction"
}

Also return a top-level consistency_score from 0 to 100, where 100 means perfectly
consistent (no contradictions found) and 0 means a politician who has directly
contradicted themselves on major issues repeatedly. Factor in the severity and
number of contradictions you found.

Return a single JSON object in this shape:
{
  "consistency_score": 0-100,
  "contradictions": [
    { "topic": ..., "quote_a": ..., "quote_a_date": ...,
      "quote_b": ..., "quote_b_date": ..., "severity": 1-3,
      "explanation": ... }
  ]
}

Only return the JSON.

STATEMENTS:
{statements}
```
